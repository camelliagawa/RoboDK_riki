# 確認・校正手順書（ポート/ボーレート・軸校正・スケール）

`dynpick_check.py`（RoboDK 不要・センサ＋PCだけで動く診断ツール）を使って、
実機で 1〜3 を順に確認します。確定した値は `force_moment_overlay.py` の冒頭パラメータに書き込みます。

前提: `pip install pyserial` 済み。センサを USB でPCに接続済み。

---

## 手順1. 接続ポート／ボーレートの確認

**目的**: `DYNPICK_PORT` と `DYNPICK_BAUD` を確定する。

1. **ポートを探す**
   ```
   python3 dynpick_check.py --list
   ```
   検出されたポートが並びます（Windows は `COM3` など、Linux は `/dev/ttyUSB0` など）。
   DynPick らしいもの（USB シリアル / FTDI 等の説明）を選びます。
   - 見分けが付かない場合は、センサの USB を**抜く→`--list`→挿す→`--list`**で、増えたポートが目的のポートです。

2. **ボーレートを自動判定する**
   ```
   python3 dynpick_check.py --port COM3 --scan-baud
   ```
   候補（921600/460800/230400/…）を順に試し、`OK raw=... 応答=...` と出たものが使える値です。
   `応答=` に**実際の1行の生文字列**も表示されるので、それを控えておいてください。

3. **確定値を書き込む** — `force_moment_overlay.py` 冒頭:
   ```python
   DYNPICK_PORT = 'COM3'      # 手順1-1 で選んだポート
   DYNPICK_BAUD = 921600      # 手順1-2 で OK だった値
   ```

> どのボーレートでも応答が取れないとき: 配線/電源を確認。応答は取れるがパースで弾かれるときは、
> `--port ... --baud ...`（scan なし）でライブ起動すると先頭に「応答サンプル(生文字列)」が出ます。
> その文字列を共有いただければ `dynpick_sensor.py` の `parse_dynpick_line()` を合わせ込みます。

## 手順2. 軸校正（AXIS_MAP と符号合わせ）

**目的**: センサ生値の並び・向きを、ツール座標（RoboDK 上の見た目）に合わせる。
「押した向き」と「＋になる軸」を実測で対応付けます。

1. **ライブ表示を起動**（起動時に無負荷で零点を自動測定 → その後の値が 0 付近から始まる）
   ```
   python3 dynpick_check.py --port COM3 --baud 921600
   ```
   `零点測定中…` の間は**センサに触れない**こと。以後 `F +0.00 +0.00 +0.00 ...` のように出ます。

2. **1方向ずつ押して観測**（力）
   - ツールの **+X 方向**に軽く押す → どの列（Fx/Fy/Fz）が動くか、符号（＋/−）はどうかを記録。
   - 同様に **+Y**, **+Z** も押して記録。
   - 表にする例:

     | 押した向き | 大きく動いた列 | 符号 |
     |---|---|---|
     | ツール +X | 例: Fx | ＋ |
     | ツール +Y | 例: Fz | − |
     | ツール +Z | 例: Fy | ＋ |

3. **`AXIS_MAP_FORCE` を作る** — 並びは `[toolX, toolY, toolZ]`、各要素 `(生値index, 符号)`。
   生値index は `Fx=0, Fy=1, Fz=2`。上の表なら:
   ```python
   #            toolX        toolY        toolZ
   AXIS_MAP_FORCE = [(0, +1), (2, -1), (1, +1)]
   #   toolX<-Fx(+)   toolY<-Fz(-)   toolZ<-Fy(+)
   ```
   - 「向きは合うが逆」の軸は符号だけ反転（例 `(0, +1)` → `(0, -1)`）。
   - 「XとYが入れ替わっている」なら index を入れ替える。

4. **モーメントも同様**に、既知方向のトルクを加えて `Mx=0, My=1, Mz=2` で `AXIS_MAP_MOMENT` を作る。

5. 書き換えたら再度ライブ表示、または RoboDK で overlay を動かして、**押した向きに矢印が伸びる**ことを確認。

> センサ取付向きが変わると対応も変わります。**取付を変えたら必ず再校正**してください。

## 手順3. スケール調整（FORCE_SCALE / MOMENT_SCALE）

**目的**: 実作業での力/モーメントの大きさに対して、矢印が見やすい長さになる倍率を決める。

1. **実作業レンジの最大値を測る** — ライブ表示のまま、**実際の研削と同程度の力**でツールを押し当て、
   一通り動かしてから `Ctrl+C` で終了。終了時に観測した最大値が出ます:
   ```
   --- 観測した最大絶対値（スケール設定の目安）---
     Fx=.. Fy=.. Fz=.. [N]
     Mx=.. My=.. Mz=.. [N*m]
     目安: FORCE_SCALE ≈ ○○ mm/N,  MOMENT_SCALE ≈ ○○ mm/(N*m)
   ```
   この「目安」は、**最大の力/モーメントで矢印が約200mm**になる倍率です。

2. **値を書き込む** — `force_moment_overlay.py` 冒頭:
   ```python
   FORCE_SCALE   = 5.0     # 目安値を参考に。矢印が短ければ大きく、長すぎれば小さく
   MOMENT_SCALE  = 300.0
   MAX_ARROW_LEN = 250.0   # これ以上は伸びない上限（振り切れ防止）
   ```
   - 矢印が**すぐ上限に張り付く**ならスケールを下げる or `MAX_ARROW_LEN` を上げる。
   - 小さすぎて**見えない**ならスケールを上げる。`FORCE_DEADBAND`/`MOMENT_DEADBAND` 未満は非表示なので、
     微小な力を見たいときはデッドバンドも下げる。

## 手順4. RoboDK 上での最終確認（本番の動かし方）

1. RoboDK を起動 → ステーションを開く → ロボットを **Run on Robot（ドライバ接続）**。
2. `force_moment_overlay.py` の冒頭パラメータ（PORT/BAUD/AXIS_MAP/SCALE）が手順1〜3の値になっていることを確認。
3. RoboDK の Python から `force_moment_overlay.py` を実行（または RoboDK 内スクリプトとして登録して実行）。
   - 起動時に `零点測定中…` が出るので、**その間ツールに何も触れない**。
4. RoboDK 側で研磨プログラム（kenma 等）を **Run**。
   - `ACTIVE_ONLY_WHEN_MOVING=True` なので、**ロボットが動いている間だけ**赤（力）・青（モーメント）矢印が TCP から出ます。
5. 押し当て方向と矢印の向き・長さを見て、必要なら手順2・3を微調整。

> まず切り分けたいとき: `USE_DEMO_SIGNAL=True` にすると、センサ無しでもダミー波形で矢印描画だけ確認できます。
> 描画が出れば RoboDK 連携はOK → あとはセンサ側（PORT/BAUD/AXIS_MAP）の問題、と切り分けられます。

## つまずいたら（チェックリスト）

| 症状 | 見るところ |
|---|---|
| ポートが出ない | USB 接続・ドライバ（FTDI 等）。`--list` を抜き挿しで比較 |
| 応答なし | `--scan-baud` でボーレート確認。配線/電源 |
| 数値は出るが変な値 | 「応答サンプル(生文字列)」を共有 → parse を調整。零点測定時に触れていないか |
| 矢印の向きが逆/入替 | `AXIS_MAP_FORCE` / `AXIS_MAP_MOMENT` の符号・index |
| 矢印が長すぎ/短すぎ | `FORCE_SCALE`/`MOMENT_SCALE`/`MAX_ARROW_LEN`/デッドバンド |
| 動作中に出ない | Run on Robot 接続か。`robot.Busy()` が反応するか。`ACTIVE_ONLY_WHEN_MOVING=False` で常時表示に切替えて切り分け |
